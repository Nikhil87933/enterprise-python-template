# Security Policy

## Supported Versions

| Version | Supported |
|----------|-----------|
| 0.1.x | ✅ |
| < 0.1 | ❌ |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** create a public GitHub issue.

Instead:

- Use GitHub's **Private Vulnerability Reporting** feature if it is enabled.
- Or contact the project maintainer directly.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

We will acknowledge reports as soon as possible and work on an appropriate fix before public disclosure.

---

## Security Practices

This project includes several security-focused development practices:

- Dependabot alerts
- Dependabot security updates
- Secret scanning
- Code scanning (CodeQL)
- Branch protection
- Pull request reviews
- Continuous Integration checks
- pre-commit hooks

---

## Disclosure Policy

Security issues will be investigated privately and fixes will be released before public disclosure whenever possible.
